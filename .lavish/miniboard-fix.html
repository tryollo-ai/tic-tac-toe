MiniBoard preview cells should never change size

In the lobby room list, each room shows a small read-only MiniBoard preview of its game. Today the cells resize based on content - empty cells collapse and cells holding an X or O grow - so the 3×3 preview looks ragged. The fix pins every cell to one fixed size, filled or empty.

2 Root cause

In components/MiniBoard/styles.module.scss:

  • .mini sets grid-template-columns: repeat(3, 1fr) but no grid-template-rows. With a fixed height: 84px but only implicit rows, each row auto-sizes to its content - so a row with a mark grows and a row of empty cells collapses.
  • .cell grid items default to min-width/height: auto, letting content push the track size and compounding the unevenness.

The main board avoids this: components/Square/styles.module.scss pins each square with aspect-ratio: 1 / 1 + width: 100%. The MiniBoard cell has no such constraint.
