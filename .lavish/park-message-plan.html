Agent loop · agent-dispatch.yml

Make the agent tell us what went wrong
- and make the run readable

Replace the hard-coded "parked for the captain" string with a real diagnosis, and surface a clean Claude Code conversation in the Actions run - without the metadata firehose that show_full_output produced.

Design source: this repo's own tokens from app/globals.scss (dark slate, cyan --accent, rose --o-color, green --win) - not a generic theme, since the plan is about this project.

1 Goal · current state · desired behavior

Today

On a no-PR finish the loop posts one fixed string: "a failed run, a risky finding it stopped on, or no change needed." It never says which. Run logs are either silent or, with show_full_output, a wall of tool-result JSON.

Goal

When a ticket parks, the issue comment should state the actual reason. And the run page should show the Claude ↔ tools conversation, readably.

Desired

A captain reading the parked issue knows why in one glance, and can open the run Summary to read the back-and-forth - no metadata, no log spelunking.

2 One parser, two payoffs

Both asks read the same artifact: claude-code-action@v1 writes an execution_file output - a JSON array of run events (system, assistant, user/tool-result, result). Parse it once, use it twice.

claude:needs-captain   ▶ Full run transcript & logs

The park step picks one of three real outcomes instead of the catch-all string:

SituationDetected byComment
PR opened no park pr_count > 0 (existing check) No park - the PR body already carries the full no-mistakes breakdown.
Ran fine, stopped without a PR park outcome == success + no PR The agent's own result text (why it stopped) + link to the run.
Run failed / timed out park outcome == failure or file missing "The run failed or timed out" + run link. On a hard timeout the file is often absent, so we degrade gracefully.

6 Failure modes I'll handle

Timeout

No/partial execution_file → generic message + run link. The existing if: always() park still fires; the ticket is never stranded.

Oversized output

Truncate the Summary; hard-cap the comment body well under GitHub's ~65k limit.

Malformed JSON

Degrade to the generic message and log the parse failure - never crash the park step.
